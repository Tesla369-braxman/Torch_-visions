## 1. Технологии

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| **Детекция** | YOLOv8 / YOLOv11 | Оптимальный баланс скорости и точности, поддержка ARM/NPU |
| **Трекинг** | BoT‑SORT (дрон) / Deep‑OC‑SORT (турель) | Удержание ID цели при перекрытиях и манёврах |
| **ReID** | OSNet (легковесный) | Повторный захват цели после потери |
| **Оптимизация** | RKNN / OpenVINO / ONNX | Квантизация INT8 для бортовых NPU |
| **Базовый слой** | OpenCV (≥ 4.8) | Чтение кадров, пред/постобработка, HUD, калибровка |
| **Языки** | Python 3.10 (ядро) + C++ (драйверы) | Скорость разработки + реальное время |
| **Конфигурация** | YAML | Гибкая настройка под разные платформы |
| **Логирование** | Redis / Kafka / файлы | Потоковая запись ошибок и телеметрии |

## 3. Обязательные требования

- [ ] Обнаружение цели за ≤ **100 мс** с момента появления в кадре
- [ ] Присвоение **уникального ID** каждой обнаруженной цели
- [ ] Передача координат (X, Y, угол, дистанция) в управляющий контроллер с частотой **≥ 10 Гц**
- [ ] Сохранение трека при пропадании цели до **3 секунд** (фильтр Калмана)
- [ ] Автоматический возврат в режим **«ПОИСК»**, если цель не появилась за **5 секунд**
- [ ] Фильтрация **статических объектов** (деревья, столбы, строения)
- [ ] Работа при **дожде, сумерках, засветке** (без переобучения, только аугментация)
- [ ] **Полная автономность** — вся логика на борту, без внешнего сервера
- [ ] Логирование **сырых кадров** при ошибках (режим «чёрного ящика»)
- [ ] Поддержка **MIPI‑CSI / USB‑камер** с частотой ≥ 30 FPS
- [ ] Обновление модели через **USB‑накопитель или OTA (Wi‑Fi)**

## 4. Сценарий работы (как пользователь видит систему)

### Старт
- На экране HUD: **сетка прицела**, компас, индикатор **«ПОИСК»**
- Алгоритм сканирует кадр, отображает рамки вокруг всех движущихся объектов

### Наведение на цель
- Пользователь кликает на цель (или цель выбирается автоматически по приоритету)
- Система фиксирует её признаки **(ReID‑вектор)**
- Переключение в режим **«ТРЕКИНГ»**
- На HUD появляется **прицельный крест**, который плавно следует за целью
- Серводвигатели отрабатывают с задержкой **< 200 мс**

### Потеря цели
- Цель уходит за препятствие / выходит из кадра
- Крест становится **оранжевым** (статус **«ТРЕК ПОТЕРЯН»**)
- Запускается таймер **5 секунд**
- Если цель не появляется → крест гаснет, система переходит в режим **«ПОИСК»**

### Возврат цели
- Цель появляется снова
- Система сравнивает текущие объекты с сохранённым ReID‑вектором
- При совпадении → **мгновенный захват** (за 1 кадр)
- Крест становится **зелёным**, наведение продолжается без участия пользователя

### Статусы HUD
| Статус | Цвет | Описание |
|--------|------|----------|
| **СКАНИРОВАНИЕ** | Синий | Активный поиск цели в кадре |
| **ЗАХВАТ** | Зелёный | Цель зафиксирована, идёт наведение |
| **ТРЕКИНГ** | Жёлтый | Устойчивое сопровождение цели |
| **ПОТЕРЯ** | Оранжевый | Цель пропала, идёт таймер ожидания |
| **ПОИСК** | Красный | Активный поиск по всей сцене |

## 5. Процесс обработки кадра: YOLO + Трекер

### 5.1. Роли компонентов

| Компонент | Что делает | Когда запускается |
|-----------|------------|-------------------|
| **YOLO (детектор)** | Находит все объекты на кадре, выдаёт координаты боксов и классы | Первым, каждый кадр |
| **Трекер** | Связывает детекции на разных кадрах, присваивает уникальные ID, прогнозирует движение | Вторым, каждый кадр (сразу после YOLO) |
| **Постобработка** | Выбирает основную цель, рассчитывает доп. параметры | Третьим, каждый кадр |
| **Управление** | Передаёт команды на сервоприводы / моторы | Четвёртым, каждый кадр |
| **HUD** | Отрисовывает интерфейс для оператора | Пятым, каждый кадр |


5.3. YOLO vs Трекер: разница
Параметр	YOLO (детектор)	Трекер
Что делает	Находит объекты на кадре	Связывает объекты между кадрами
Выдаёт	Боксы без ID: [x1,y1,x2,y2,conf]	Треки с ID: [ID, x, y, w, h, conf]
Запоминает историю	Нет	Да (фильтр Калмана)
Прогнозирует движение	Нет	Да
Без него	Рамки "прыгают" от кадра к кадру	Не с чем работать
5.4. Реализация на разных платформах
Платформа	YOLO	Трекер
Mac M1	PyTorch / MLX / ONNX	Python (BoT‑SORT, ByteTrack)
Jetson	TensorRT (экспорт из ONNX)	Python или C++
CAM-1126BQ38	RKNN (конвертация из ONNX)	Python (оптимизированный)
Orange Pi 5	RKNN (конвертация из ONNX)	Python (оптимизированный)
Важно: YOLO всегда запускается на NPU (TensorRT / RKNN), трекер — на CPU. Это стандартная практика.

**Самый свежий и релевантный вариант.**

## 2. Модели для детекции дронов и птиц

### Drone Detector (YOLOv8)
**Высокоточная модель для различения дронов и птиц.**

- **Классы:** Drone, Bird, UAV, Airplane
- **Точность:** **mAP50 = 96.1%** [citation:1]
- **Precision:** 94.1%
- **Recall:** 94.1%
- **Архитектура:** YOLOv8
- **Где взять:** [GitHub - DeepPythonist/drone-detector](https://github.com/DeepPythonist/drone-detector)

> **Применение:** Идеально для турели, где критично отличать дрон от птицы (снижение ложных срабатываний). Модель создана специально для систем ПВО и защиты критической инфраструктуры [citation:1].

---

### Drone-Military-Detection-YOLO
**Проект с обучением на военном датасете дронов.**

- **Архитектура:** YOLOv8
- **Особенность:** Готовый пайплайн для тренировки на Roboflow/Kaggle датасете, расчёт отклонения цели от центра кадра (для наведения) [citation:2].
- **Где взять:** [GitHub - chitransha12/Drone-Military-Detection-YOLO](https://github.com/chitransha12/Drone-Military-Detection-YOLO)

> **Применение:** Полезен не только готовой моделью, но и кодом для интеграции с системой наведения (расчёт dx, dy, дистанции до центра кадра) [citation:2].

### Adaptive Lightweight YOLOv5 Military Target Detection
**Лёгкая модель для работы в сложных погодных условиях.**

- **Архитектура:** YOLOv5 (легковесная)
- **Классы:** Военная техника, солдаты, ракеты, дроны [citation:3]
- **Условия:** Туман, дым, ночное видение, плохое освещение
- **Где взять:** [GitHub - shashankbs2003/Adaptive-Lightweight-YOLOv5-Military-Target-Detection](https://github.com/shashankbs2003/Adaptive-Lightweight-YOLOv5-Military-Target-Detection)

> **Применение:** Для работы на ресурсо-ограниченных устройствах (дроны, Edge) в условиях плохой видимости [citation:3].

## 3. Специализированные исследовательские модели

### IMS-YOLOv8
**Лёгкая модель для детекции 7 типов дронов и птиц.**

- **Архитектура:** YOLOv8n (модифицированная)
- **Параметры:** **1.73 М** (на 42.6% меньше базового YOLOv8n) [citation:11]
- **Классы:** 7 моделей дронов + птицы
- **Особенность:** Оптимизирована для Edge-устройств (Jetson, Raspberry Pi) [citation:11]
- **Где взять:** Статья в IEEE Xplore (код ожидается в репозитории авторов)

> **Применение:** Для перехватчика, которому нужно различать типы дронов, при этом модель должна быть лёгкой для работы на встраиваемой плате [citation:11].

---

### YOLO-RAW
**Модель для детекции дронов в плохую погоду.**

- **Архитектура:** YOLO (модифицированная)
- **Устойчивость:** Дождь, шум, Motion Blur (смаз от движения) [citation:4]
- **Где взять:** [GitHub - AdnanMunir338/YOLO-RAW](https://github.com/AdnanMunir338/YOLO-RAW)

> **Применение:** Для работы на открытом воздухе при осадках или в условиях ухудшения качества изображения (дрон на высокой скорости) [citation:4].

## Как использовать

После скачивания файла `.pt` используйте библиотеку `ultralytics`:

```python
from ultralytics import YOLO

# Загрузка модели из скачанного файла
model = YOLO('путь/к/вашей/модели/yolov8m_defence.pt')

# Запуск детекции на изображении
results = model('путь/к/изображению.jpg')
results[0].show()  # Показать результат
