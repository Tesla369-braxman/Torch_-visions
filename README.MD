# Техническое задание
## Система машинного зрения для дрона/турели
### (ARM / RISC‑V экосистема, бюджетные решения)

---

## 1. Технологии

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| **Детекция** | YOLOv8 / YOLOv11 | Оптимальный баланс скорости и точности, поддержка ARM/NPU |
| **Трекинг** | BoT‑SORT (дрон) / Deep‑OC‑SORT (турель) | Удержание ID цели при перекрытиях и манёврах |
| **ReID** | OSNet (легковесный) | Повторный захват цели после потери |
| **Оптимизация** | RKNN / OpenVINO / ONNX | Квантизация INT8 для бортовых NPU |
| **Базовый слой** | OpenCV (≥ 4.8) | Чтение кадров, пред/постобработка, HUD, калибровка |
| **Языки** | Python 3.10 (ядро) + C++ (драйверы) | Скорость разработки + реальное время |
| **Конфигурация** | YAML | Гибкая настройка под разные платформы |
| **Логирование** | Redis / Kafka / файлы | Потоковая запись ошибок и телеметрии |

## 2. Примеры реализованных решений

| Задача | Решение | Результат |
|--------|---------|-----------|
| **Перехват FPV-дрона** | YOLOv8n + BoT‑SORT на **RV1126B** | 38 FPS, захват за 80 мс |
| **Антидронная турель** | Две камеры (ширик + зум), переключение моделей | Обнаружение с 150 м на **Orange Pi 5** |

---

## 3. Обязательные требования

- [ ] Обнаружение цели за ≤ **100 мс** с момента появления в кадре
- [ ] Присвоение **уникального ID** каждой обнаруженной цели
- [ ] Передача координат (X, Y, угол, дистанция) в управляющий контроллер с частотой **≥ 10 Гц**
- [ ] Сохранение трека при пропадании цели до **3 секунд** (фильтр Калмана)
- [ ] Автоматический возврат в режим **«ПОИСК»**, если цель не появилась за **5 секунд**
- [ ] Фильтрация **статических объектов** (деревья, столбы, строения)
- [ ] Работа при **дожде, сумерках, засветке** (без переобучения, только аугментация)
- [ ] **Полная автономность** — вся логика на борту, без внешнего сервера
- [ ] Логирование **сырых кадров** при ошибках (режим «чёрного ящика»)
- [ ] Поддержка **MIPI‑CSI / USB‑камер** с частотой ≥ 30 FPS
- [ ] Обновление модели через **USB‑накопитель или OTA (Wi‑Fi)**

## 4. Сценарий работы (как пользователь видит систему)

### Старт
- На экране HUD: **сетка прицела**, компас, индикатор **«ПОИСК»**
- Алгоритм сканирует кадр, отображает рамки вокруг всех движущихся объектов

### Наведение на цель
- Пользователь кликает на цель (или цель выбирается автоматически по приоритету)
- Система фиксирует её признаки **(ReID‑вектор)**
- Переключение в режим **«ТРЕКИНГ»**
- На HUD появляется **прицельный крест**, который плавно следует за целью
- Серводвигатели отрабатывают с задержкой **< 200 мс**

### Потеря цели
- Цель уходит за препятствие / выходит из кадра
- Крест становится **оранжевым** (статус **«ТРЕК ПОТЕРЯН»**)
- Запускается таймер **5 секунд**
- Если цель не появляется → крест гаснет, система переходит в режим **«ПОИСК»**

### Возврат цели
- Цель появляется снова
- Система сравнивает текущие объекты с сохранённым ReID‑вектором
- При совпадении → **мгновенный захват** (за 1 кадр)
- Крест становится **зелёным**, наведение продолжается без участия пользователя

### Статусы HUD
| Статус | Цвет | Описание |
|--------|------|----------|
| **СКАНИРОВАНИЕ** | Синий | Активный поиск цели в кадре |
| **ЗАХВАТ** | Зелёный | Цель зафиксирована, идёт наведение |
| **ТРЕКИНГ** | Жёлтый | Устойчивое сопровождение цели |
| **ПОТЕРЯ** | Оранжевый | Цель пропала, идёт таймер ожидания |
| **ПОИСК** | Красный | Активный поиск по всей сцене |

## 5. Целевые платформы (бюджетные, ARM / RISC‑V)

| Категория | Платформа | NPU (TOPS) | FPS (YOLOv8n) | Применение | Цена (прим.) |
|-----------|-----------|------------|---------------|------------|--------------|
| **⬇️ Демо / прототип** | Pi Zero 2W / 3W | 0 | 2–7 | Отладка логики | ~$15–25 |
| **✅ Дрон (лёгкий)** | **RV1126B** | **3** | **~38** | Дрон, охрана периметра | ~$40–60 |
| **✅ Дрон (универсальный)** | Orange Pi 5 | 6 | ~53 | Дрон / турель | ~$100–150 |
| **✅ Турель (бюджетная)** | RDK X5 | 10 | ~55 | Стационарная турель | ~$200–300 |
| **⚡ Турель (мощная)** | Orange Pi 6 Plus | ~30 | ~80–100+ | Тяжёлая турель, Edge AI | ~$250–350 |

## 6. Рекомендации по выбору платформы

| Сценарий | Рекомендованная платформа | Причина |
|----------|---------------------------|---------|
| **Дрон (лёгкий, бюджетный)** | **RV1126B** | 38 FPS, низкое энергопотребление, малый вес, MIPI‑CSI |
| **Дрон (универсальный)** | Orange Pi 5 | 53 FPS, больше памяти, гибкость |
| **Турель (бюджетная)** | Orange Pi 5 / RDK X5 | Высокий FPS за небольшие деньги |
| **Турель (профессиональная)** | Orange Pi 6 Plus | Запас мощности под тяжёлые модели |
| **Прототип / демо** | Pi Zero 3W | Минимальная цена, только для отладки логики |

## 7. Интерфейсы ввода‑вывода

**Вход:**
- MIPI‑CSI (для камер)
- USB Video Class (UVC)
- RTSP (IP‑камеры)
- UART / I²C (для дальномеров, IMU, энкодеров)

**Выход:**
- UART / PWM (управление сервоприводами)
- REST API (логи, телеметрия)
- HDMI (HUD‑дисплей)

## 8. Этапы разработки

1. **Сбор и разметка датасета** (10 000+ кадров, COCO‑формат)
2. **Обучение базовой модели** (YOLOv8, эксперименты с архитектурой)
3. **Интеграция трекера** (BoT‑SORT / Deep‑OC‑SORT)
4. **Оптимизация** (квантизация INT8, конвертация в RKNN / ONNX)
5. **Разработка HUD и интерфейса** (OpenCV + PyQt)
6. **Интеграция с железом** (драйверы камеры, сервоприводов, датчиков)
7. **Полевые испытания и дообучение**
8. **Упаковка в Docker и документация**

## 9. Итоговый вывод

Разрабатываемая система представляет собой **универсальное, бюджетное и масштабируемое решение** для задач наведения на подвижные цели.  
Она построена на **ARM / RISC‑V экосистеме**, использует **открытые технологии** (YOLO, OpenCV, RKNN) и **не привязана к дорогим платформам**.

**Рекомендуемая стартовая платформа:** **RV1126B** — для дронов и лёгких турелей.  
**Для мощных турелей:** **Orange Pi 6 Plus** — с запасом производительности под любые модели.
