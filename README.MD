# Техническое задание: Система машинного зрения для дрона/турели

## 1. Технологический стек

| Компонент | Технология | Примечание |
|-----------|------------|------------|
| Детекция | YOLOv8 / YOLOv11 | Баланс скорости и точности |
| Трекинг | BoT-SORT (дрон) / Deep-OC-SORT (турель) | Удержание ID цели |
| ReID | OSNet (легковесный) | Для возврата цели после потери |
| Оптимизация | TensorRT / OpenVINO | INT8/FP16 для бортовых систем |
| Языки | Python 3.10 + C++ | Python — инференс, C++ — драйверы |
| Визуализация | OpenCV + PyQt/QML | HUD-интерфейс |
| Логирование | Redis / Kafka | Потоковая запись ошибок |
| Конфигурация | YAML | Гибкая настройка параметров |

## 3. Обязательные требования (что должно выполняться)

- [ ] Обнаружение цели за ≤ 100 мс с момента появления
- [ ] Присвоение уникального ID каждой цели
- [ ] Передача координат (X, Y, угол, дистанция) с частотой ≥ 10 Гц
- [ ] Сохранение трека при пропадании цели до 3 секунд (фильтр Калмана)
- [ ] Автоматический возврат в режим поиска через 5 секунд без цели
- [ ] Фильтрация статических объектов (деревья, столбы)
- [ ] Работа при дожде, сумерках, засветке (без переобучения)
- [ ] Полная автономность: вся логика на борту, без сервера
- [ ] Логгирование сырых кадров при ошибках (чёрный ящик)

## 4. Сценарий работы (как видит пользователь)

### Старт
- На экране HUD: сетка прицела, компас, индикатор «ПОИСК»
- Алгоритм сканирует кадр, показывает рамки вокруг движущихся объектов

### Наведение на цель
- Пользователь кликает на цель → система фиксирует её признаки (ReID-вектор)
- Переключение в режим трекинга (оптимизированная модель)
- На HUD появляется прицельный крест, который плавно следует за целью
- Серводвигатели отрабатывают с задержкой < 200 мс

### Потеря цели
- Цель уходит за препятствие
- Крест становится оранжевым (статус «ТРЕК ПОТЕРЯН»)
- Таймер обратного отсчёта 5 секунд
- Если цель не появляется → крест гаснет, переход в режим «ПОИСК»

### Возврат цели
- Цель появляется снова
- Система сравнивает текущие объекты с сохранённым ReID-вектором
- При совпадении → мгновенный захват (за 1 кадр)
- Крест становится зелёным, наведение продолжается

### Статусы HUD
| Статус | Цвет | Описание |
|--------|------|----------|
| СКАНИРОВАНИЕ | Синий | Поиск цели в кадре |
| ЗАХВАТ | Зелёный | Цель зафиксирована |
| ТРЕКИНГ | Жёлтый | Сопровождение цели |
| ПОТЕРЯ | Оранжевый | Цель пропала, идёт таймер |
| ПОИСК | Красный | Активный поиск по сцене |

## 5. Критерии приёмки

- [ ] При тестовом пролёте/повороте турели цель теряется не более 2 раз за 10 минут
- [ ] Время повторного захвата после потери < 2 секунд
- [ ] Случайная смена ID цели (переключение на другой объект) = 0 за сессию
- [ ] Интерфейс корректно отображает все статусы
- [ ] Система работает при освещённости от 0.1 до 100 000 люкс
- [ ] Журнал ошибок сохраняет последние 1000 кадров
- [ ] Обновление модели через USB или Wi-Fi (OTA)

## 7. Интерфейсы ввода-вывода

**Вход:**
- RTSP-поток (IP-камеры)
- USB Video Class (UVC)
- MIPI-CSI (для одноплатников)

**Выход:**
- UART (для управления сервоприводами/моторами)
- REST API (для логов и телеметрии)
- HDMI (для HUD-дисплея)
